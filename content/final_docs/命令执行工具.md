### exec_command

后台shell命令执行工具，执行目标命令，最多等待yield_time_ms 时间后进程没有执行完则收集进程的当前输出返回给模型，后续用 write_stdin 通过 session_id 继续操作它。如果进程已经执行完会提前返回并附带exit_code。

需要注意的是，由于启动的是后台进程，除了到达yield_time 之后主动给模型发送一次后台任务输出之外，不会再给模型任何提示，需要模型自己记得"我还有一个后台进程在跑着呢"。

exec_command 暴露的主要参数：

```json
{
  "cmd": "npm run dev",
  "workdir": "D:\\work\\codex",
  "shell": "powershell",
  "tty": true,
  "yield_time_ms": 1000,
  "max_output_tokens": 4000,
  "login": false,
  "environment_id": "primary",
  "sandbox_permissions": "use_default",
  "additional_permissions": {
    "network": { "enabled": true },
    "file_system": {
      "read": ["D:\\somewhere"],
      "write": ["D:\\somewhere"]
    }
  },
  "justification": "需要联网安装依赖",
  "prefix_rule": ["npm", "install"]
}
```

- cmd: 要执行的 shell 命令。
- workdir: 工作目录。
- shell: 指定 shell binary，默认用用户 shell(powershell)。
- tty: 是否分配 PTY。要后续 write_stdin 交互，通常需要 tty: true。
- yield_time_ms: 等多久先把已有输出返回给模型，不等同于老工具的总超时。
- max_output_tokens: 本次输出 token 上限。
- login: 只有配置允许 login shell 时才可用。若配置不允许但传 true，会直接报错。
- environment_id: 多环境模式下才暴露。
- sandbox_permissions： 权限策略，常见值是 use_default、require_escalated，有特性开关时还可能有 with_additional_permissions。但这不是决定最后审批操作的直接原因。
- justification：需要提权时展示给用户的理由。
- prefix_rule：可复用的审批前缀，比如 ["git", "pull"]。
- additional_permissions: 细粒度申请文件系统/网络权限，只在对应特性启用时使用。

返回当前输出和session_id(命令没完)

### write_stdin

只用来操作exec_command 的未完命令，参数如下：

```
{
  "session_id": 12345,
  "chars": "y\n",
  "yield_time_ms": 1000,
  "max_output_tokens": 4000
}
```

chars为空则只是查询最新返回消息。返回值跟exec_command一样。

### shell_command

一次性命令工具。启动一条 shell 命令，等它结束或超时，然后把输出返回给模型。它不会保存进程，也不会返回 session_id。

shell_command 暴露的主要参数：

```json
{
  "command": "Get-ChildItem -Force",
  "workdir": "D:\\work\\codex",
  "timeout_ms": 10000,
  "login": false,
  "sandbox_permissions": "use_default",
  "additional_permissions": {
    "network": { "enabled": true },
    "file_system": {
      "read": ["D:\\somewhere"],
      "write": ["D:\\somewhere"]
    }
  },
  "justification": "需要访问额外路径",
  "prefix_rule": ["git", "pull"]
}
```

- command: 要执行的 shell script。
- workdir: 工作目录。
- timeout_ms: 最大运行时间，默认 10000 ms。
- login: 同上
- 同样有权限相关参数。

返回命令结果。

- 如果输出太长，会按 turn 的 truncation policy 截断。
- 超时就返回超时

读代码主要靠这个工具能力

```
rg -n "build_tools" .
Get-Content path\to\file.rs
```
